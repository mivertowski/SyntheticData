## Tier 3 — Counterfactual Engine Extension

### 3.1 Extend causal DAG with audit/financial nodes
Add 12 new nodes to `causal_dag_default.yaml`:
- `materiality_threshold` → bound to `audit_standards.sox.materiality_threshold`
- `inherent_risk` → bound to `anomaly_injection.rates.total_rate`
- `control_risk` → derived from `control_effectiveness`
- `combined_risk` → derived from `inherent_risk` × `control_risk`
- `sample_size_factor` → derived from `combined_risk`
- `misstatement_detection_rate` → derived from `sample_size_factor` × `control_effectiveness`
- `opinion_severity` → derived from `misstatement_detection_rate` (output only)
- `gross_margin` → bound to `distributions.amounts.components[1].weight`
- `net_income_ratio` → derived from `revenue_growth` - expenses
- `debt_ratio` → bound to `balance.opening_balance_debt_to_equity`
- `ecl_provision_rate` → derived from `bad_debt_rate`
- `going_concern_risk` → derived from `debt_ratio` + `net_income_ratio`

### 3.2 Add config bindings for new DAG nodes
Wire each node's `config_binding` to actual `GeneratorConfig` fields

### 3.3 Map intervention types to new nodes
- `RegulatoryChange::MaterialityThresholdChange` → `materiality_threshold`
- `RegulatoryChange::AuditStandardChange` → `inherent_risk`, `sample_size_factor`
- `RegulatoryChange::TaxRateChange` → `tax_rate` (new node)
- `MacroShock::CreditCrunch` → `ecl_provision_rate`, `bad_debt_rate`, `going_concern_risk`

### 3.4 Add audit-specific counterfactual specs to Engine A
- `MutateMateriality(factor)` → scales materiality and re-derives sampling
- `MutateCRA(account, new_level)` → overrides CRA for an account area
- `MutateGoingConcern(conclusion)` → forces a GC conclusion and re-derives opinion
- `RemediateFinding(finding_id)` → marks a finding as remediated

### 3.5 Create audit scenario packs
- `audit_scope_change.yaml` — reduces materiality by 30%, observe cascade to sampling/findings/opinion
- `control_failure_cascade.yaml` — increases control_risk for revenue, observe CRA→sampling→misstatement chain
- `going_concern_trigger.yaml` — injects working capital deficiency + debt covenant breach
- `fraud_detection_improvement.yaml` — increases control_effectiveness, observe reduced misstatement_rate

### 3.6 Add counterfactual propagation through audit chain
After a counterfactual intervention, re-run:
1. Materiality calculation (if materiality changed)
2. CRA assignment (if risk levels changed)
3. Sampling plan generation (if CRA changed)
4. Finding evaluation (if sampling changed)
5. Opinion derivation (if findings changed)

### 3.7 Wire counterfactual pairs output for audit entities
Currently only JE pairs are exported. Add support for:
- (original_opinion, counterfactual_opinion) pairs
- (original_cra, counterfactual_cra) pairs
- Full audit chain diff: original chain vs counterfactual chain

## Cross-Cutting Concerns

### Testing Strategy
- Each fix gets a unit test verifying the fix
- Integration test for the full audit causal chain (CRA→sampling→findings→opinion)
- Sequential per-crate testing only (`--test-threads=1`)

### Backward Compatibility
- All fixes are backward-compatible (no config schema changes)
- New causal DAG nodes are additive (existing DAGs still work)
- New counterfactual specs are additive (existing specs unchanged)

### Performance
- No expected performance impact from fixes
- Causal DAG propagation adds <1ms per intervention
- Audit chain re-run adds <10ms per counterfactual scenario
