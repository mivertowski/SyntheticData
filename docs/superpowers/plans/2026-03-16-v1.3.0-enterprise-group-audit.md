# v1.3.0 Enterprise Group Audit — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform DataSynth into a comprehensive enterprise group audit simulation platform covering ISA, IFRS, US GAAP, and local regulations — 32 items across 5 implementation tiers.

**Architecture:** Each tier builds on the previous and leaves the codebase in a releasable state. New models go in `datasynth-core/src/models/`, generators in `datasynth-generators/src/`, orchestration in `datasynth-runtime/src/enhanced_orchestrator.rs`, and output serialization in `datasynth-cli/src/output_writer.rs`. All features are config-gated with `enabled: bool`.

**Tech Stack:** Rust workspace (16 crates), ChaCha8 deterministic RNG, rust_decimal for precise amounts, chrono for dates, serde for serialization

**Spec:** `docs/superpowers/specs/2026-03-16-v1.3.0-enterprise-group-audit-design.md`

**Plan structure:** Tier 0 has full TDD steps with code. Tiers 1-4 have task-level breakdowns with file paths and acceptance criteria — detailed TDD steps are added per-tier when execution reaches them.

```rust
use datasynth_core::models::documents::payment::{Payment, PaymentType};
use datasynth_core::models::subledger::ap::APInvoice;
use datasynth_core::models::subledger::ar::ARInvoice;
use rust_decimal::Decimal;
use std::collections::HashMap;

/// Apply AP payments to reduce APInvoice.amount_remaining.
/// Matches payments to invoices by invoice reference in payment allocations.
pub fn apply_ap_settlements(ap_invoices: &mut [APInvoice], payments: &[Payment]) {
    // Build lookup: invoice_number -> indices in ap_invoices
    // Build lookup by vendor_invoice_number (the document flow invoice ID),
    // NOT by invoice_number (internal AP invoice number).
    let mut invoice_map: HashMap<&str, Vec<usize>> = HashMap::new();
    for (idx, inv) in ap_invoices.iter().enumerate() {
        invoice_map.entry(inv.vendor_invoice_number.as_str()).or_default().push(idx);
    }

    for payment in payments {
        if payment.payment_type != PaymentType::ApPayment {
            continue;
        }
        // PaymentAllocation.invoice_id maps to the document flow's vendor invoice ID,
        // which corresponds to APInvoice.vendor_invoice_number (NOT invoice_number).
        for alloc in &payment.allocations {
            if let Some(indices) = invoice_map.get(alloc.invoice_id.as_str()) {
                for &idx in indices {
                    let inv = &mut ap_invoices[idx];
                    let settlement = alloc.amount.min(inv.amount_remaining);
                    inv.amount_remaining = (inv.amount_remaining - settlement).max(Decimal::ZERO);
                }
            }
        }
    }
}

/// Apply AR receipts to reduce ARInvoice.amount_remaining.
/// Matches receipts to invoices by invoice reference in payment allocations.
pub fn apply_ar_settlements(ar_invoices: &mut [ARInvoice], payments: &[Payment]) {
    // Build lookup by customer_invoice_number (the document flow invoice ID).
    let mut invoice_map: HashMap<&str, Vec<usize>> = HashMap::new();
    for (idx, inv) in ar_invoices.iter().enumerate() {
        invoice_map.entry(inv.invoice_number.as_str()).or_default().push(idx);
    }

    for payment in payments {
        if payment.payment_type != PaymentType::ArReceipt {
            continue;
        }
        for alloc in &payment.allocations {
            if let Some(indices) = invoice_map.get(alloc.invoice_id.as_str()) {
                for &idx in indices {
                    let inv = &mut ar_invoices[idx];
                    let settlement = alloc.amount.min(inv.amount_remaining);
                    inv.amount_remaining = (inv.amount_remaining - settlement).max(Decimal::ZERO);
                }
            }
        }
    }
}
```

```rust
//! Converts EliminationEntry records to JournalEntry records for GL integration.

use datasynth_core::models::intercompany::elimination::EliminationEntry;
use datasynth_core::models::journal_entry::{JournalEntry, JournalEntryHeader};

/// Convert elimination entries to journal entries for the consolidated GL.
pub fn elimination_to_journal_entries(entries: &[EliminationEntry]) -> Vec<JournalEntry> {
    entries.iter().map(elimination_to_je).collect()
}

fn elimination_to_je(entry: &EliminationEntry) -> JournalEntry {
    let mut header = JournalEntryHeader::new(
        entry.consolidation_entity.clone(),
        entry.entry_date,
    );
    header.document_type = "ELIMINATION".to_string();
    header.reference = Some(entry.entry_id.clone());
    header.header_text = Some(entry.description.clone());
    header.is_elimination = true;
    header.created_by = "CONSOLIDATION".to_string();

    let mut je = JournalEntry::new(header);
    // Use JournalEntryLine::debit() / ::credit() — NOT a with_line() builder
    // Signature: JournalEntryLine::debit(document_id, line_number, gl_account, amount)
    for (i, line) in entry.lines.iter().enumerate() {
        let je_line = if line.is_debit {
            JournalEntryLine::debit(
                je.header.document_id,
                (i + 1) as u32,
                line.account.clone(),
                line.amount,
            )
        } else {
            JournalEntryLine::credit(
                je.header.document_id,
                (i + 1) as u32,
                line.account.clone(),
                line.amount,
            )
        };
        je.add_line(je_line);
    }
    je
}
```

```rust
/// Phase: Period Close — runs depreciation, accruals, tax provision stub,
/// income statement close, and retained earnings rollforward.
fn phase_period_close(
    &mut self,
    entries: &[JournalEntry],
    coa: &ChartOfAccounts,
    companies: &[CompanyConfig],
) -> SynthResult<Vec<JournalEntry>> {
    info!("Phase: Period Close");

    let mut close_entries = Vec::new();

    for company in companies {
        // Income statement close: zero out revenue and expense accounts
        // DR Revenue accounts, CR Income Summary
        // DR Income Summary, CR Expense accounts
        // DR/CR Income Summary to Retained Earnings

        let period_end = self.get_period_end_date();

        // Calculate net income from existing entries for this company
        let (total_revenue, total_expenses) = self.calculate_revenue_and_expenses(
            entries, &company.code, period_end,
        );

        let net_income = total_revenue - total_expenses;

        // Tax provision stub: statutory_rate * pre_tax_income
        let statutory_rate = rust_decimal_macros::dec!(0.21); // US default
        let tax_expense = net_income * statutory_rate;

        if tax_expense > Decimal::ZERO {
            let mut tax_header = JournalEntryHeader::new(
                company.code.clone(),
                period_end,
            );
            tax_header.document_type = "TAX_PROVISION".to_string();
            tax_header.header_text = Some("Period-end tax provision".to_string());

            let tax_je = JournalEntry::new(tax_header)
                .with_line("7100".to_string(), tax_expense, true)  // DR Tax Expense
                .with_line("2300".to_string(), tax_expense, false); // CR Tax Payable

            close_entries.push(tax_je);
        }

        // Closing entry: DR Revenue, CR Income Summary, DR Income Summary, CR Expenses
        // Then DR/CR Income Summary to Retained Earnings
        // (Simplified: single closing entry transferring net income to retained earnings)
        let mut close_header = JournalEntryHeader::new(
            company.code.clone(),
            period_end,
        );
        close_header.document_type = "CLOSING".to_string();
        close_header.header_text = Some("Income statement close to retained earnings".to_string());

        // Net income → retained earnings
        if net_income > Decimal::ZERO {
            let close_je = JournalEntry::new(close_header)
                .with_line("3900".to_string(), net_income, false); // CR Retained Earnings
            // Note: debit side is the income summary which nets to zero
            // This is simplified — full implementation should DR each revenue account
            // and CR each expense account individually
            close_entries.push(close_je);
        }
    }

    Ok(close_entries)
}
```

```rust
use datasynth_core::models::balance::opening_balance::GeneratedOpeningBalance;
use datasynth_core::models::journal_entry::{JournalEntry, JournalEntryHeader, JournalEntryLine};
use datasynth_core::models::ChartOfAccounts;
use rust_decimal::Decimal;

/// Convert opening balances to journal entries.
///
/// IMPORTANT: Uses the ChartOfAccounts to look up each account's type to determine
/// the correct debit/credit side. This handles contra-asset accounts (e.g., Accumulated
/// Depreciation starts with '1' but is credit-normal) correctly.
///
/// The existing orchestrator comment (line 1625-1629) documents why a prefix-only
/// heuristic is insufficient — this function solves that by using CoA account types.
pub fn opening_balance_to_jes(
    ob: &GeneratedOpeningBalance,
    coa: &ChartOfAccounts,
) -> Vec<JournalEntry> {
    if ob.balances.is_empty() {
        return Vec::new();
    }

    let mut header = JournalEntryHeader::new(
        ob.company_code.clone(),
        ob.as_of_date,
    );
    header.document_type = "OPENING_BALANCE".to_string();
    header.header_text = Some("Opening balance".to_string());
    header.created_by = "SYSTEM".to_string();

    let mut je = JournalEntry::new(header);
    let mut line_number = 1u32;

    for (account, amount) in &ob.balances {
        if *amount == Decimal::ZERO {
            continue;
        }

        // Look up account type from CoA to determine normal balance side.
        // Assets and Expenses are debit-normal; Liabilities, Equity, and Revenue
        // are credit-normal. Contra accounts reverse the normal side.
        let is_debit = match coa.get_account(account) {
            Some(gl_account) => gl_account.is_debit_normal(),
            None => {
                // Fallback: use prefix heuristic if account not in CoA
                // 1xxx = Asset (debit), 2xxx = Liability (credit), 3xxx = Equity (credit)
                account.starts_with('1') || account.starts_with('5') || account.starts_with('6')
            }
        };

        let abs_amount = amount.abs();
        let je_line = if is_debit {
            JournalEntryLine::debit(je.header.document_id, line_number, account.clone(), abs_amount)
        } else {
            JournalEntryLine::credit(je.header.document_id, line_number, account.clone(), abs_amount)
        };
        je.add_line(je_line);
        line_number += 1;
    }

    vec![je]
}
```

Note: The function takes a `&ChartOfAccounts` parameter to resolve the contra-asset problem
documented in the existing orchestrator code. The `is_debit_normal()` method (or equivalent)
must be checked on `GLAccount` — if it doesn't exist, add it based on `AccountType`.

Ensure the function is exported through the module's public API. The generators' `balance/mod.rs`
re-exports via `pub use opening_balance_generator::*;` — add the new function to that module
or create a new file `opening_balance_converter.rs` and add `pub use opening_balance_converter::*;`.
